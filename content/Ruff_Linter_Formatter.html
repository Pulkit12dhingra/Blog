Ruff: The Blazingly Fast Python Linter and Formatter

← Back to Home

If you're tired of slow linting tools and want a single tool that does it all, Ruff is about to become your new best friend. Written in Rust, Ruff is an extremely fast Python linter and code formatter that can replace Flake8, Black, isort, pyupgrade, and more — all while being 10-100x faster.

What is Ruff?

Ruff is an all-in-one Python linting and formatting tool built for speed and comprehensiveness. It implements over 700 lint rules from popular tools like:

  • Flake8 and its plugins
  • pylint
  • pycodestyle
  • pyflakes
  • isort (import sorting)
  • pydocstyle (docstring checks)
  • pyupgrade (syntax modernization)
  • And many more...

Why Choose Ruff?

  • Speed: 10-100x faster than traditional linters (written in Rust)
  • All-in-One: Combines linting, formatting, and import sorting
  • 700+ Rules: Comprehensive checks covering style, bugs, and best practices
  • Auto-Fix: Automatically fixes many issues with --fix
  • IDE Integration: Works with VS Code, PyCharm, Vim, and more
  • Drop-in Replacement: Compatible with existing configurations

Installation

Install Ruff using pip or UV:

# Using pip
pip install ruff

# Using UV (even faster!)
uv pip install ruff

# Or add to your project
uv add ruff --dev

Basic Usage

Linting Your Code

Run Ruff on your Python files or directories:

# Lint a single file
ruff check script.py

# Lint entire directory
ruff check .

# Lint with auto-fix
ruff check . --fix

# Show all violations (including fixed)
ruff check . --fix --show-fixes
Formatting Your Code

Ruff includes a formatter compatible with Black:

# Format files
ruff format .

# Check formatting without changes
ruff format . --check

# Format specific file
ruff format script.py
Running Both Together
# Lint and format in one go
ruff check . --fix && ruff format .

Configuration

Configure Ruff using pyproject.toml, ruff.toml, or .ruff.toml:

Basic Configuration (pyproject.toml)
[tool.ruff]
# Set line length
line-length = 88

# Enable specific rule sets
select = [
    "E",   # pycodestyle errors
    "W",   # pycodestyle warnings
    "F",   # pyflakes
    "I",   # isort
    "B",   # flake8-bugbear
    "C4",  # flake8-comprehensions
    "UP",  # pyupgrade
]

# Ignore specific rules
ignore = ["E501"]  # Line too long

# Exclude directories
exclude = [
    ".git",
    ".venv",
    "__pycache__",
    "build",
    "dist",
]

# Fix automatically when possible
fix = true

[tool.ruff.format]
# Use double quotes
quote-style = "double"

# Indent with spaces
indent-style = "space"

Rule Categories

Ruff organizes rules into categories. Here are the most common ones:

Code Category Description
E pycodestyle errors Style violations and errors
W pycodestyle warnings Style warnings
F Pyflakes Logical errors, unused imports
I isort Import sorting and organization
N pep8-naming Naming conventions
D pydocstyle Docstring conventions
UP pyupgrade Upgrade syntax to modern Python
B flake8-bugbear Bug and design problems
A flake8-builtins Shadowing Python builtins
C4 flake8-comprehensions List/dict comprehension issues
SIM flake8-simplify Code simplification suggestions
RUF Ruff-specific Ruff's own rules

Common Use Cases

1. Replacing Flake8 + Black + isort

Before (multiple tools):

flake8 .
black .
isort .

After (just Ruff):

ruff check . --fix
ruff format .
2. Pre-commit Hook

Add to .pre-commit-config.yaml:

repos:
  - repo: https://github.com/astral-sh/ruff-pre-commit
    rev: v0.1.9
    hooks:
      # Run the linter
      - id: ruff
        args: [--fix]
      # Run the formatter
      - id: ruff-format
3. CI/CD Integration

GitHub Actions example:

- name: Install Ruff
  run: pip install ruff

- name: Run Ruff linter
  run: ruff check .

- name: Run Ruff formatter check
  run: ruff format . --check
4. VS Code Integration

Install the Ruff extension and add to settings.json:

{
  "[python]": {
    "editor.formatOnSave": true,
    "editor.codeActionsOnSave": {
      "source.fixAll": true,
      "source.organizeImports": true
    },
    "editor.defaultFormatter": "charliermarsh.ruff"
  }
}

Advanced Features

Ignoring Violations

Ignore specific lines or blocks:

# Ignore entire file
# ruff: noqa

# Ignore specific line
x = some_long_function_call()  # noqa: E501

# Ignore specific rule on line
import os  # noqa: F401

# Ignore multiple rules
result = eval(user_input)  # noqa: S307, E501
Per-File Configuration
[tool.ruff]
[tool.ruff.per-file-ignores]
"__init__.py" = ["F401"]  # Unused imports OK in __init__
"tests/*" = ["S101"]      # Assert statements OK in tests
Custom Import Order
[tool.ruff.isort]
known-first-party = ["myapp"]
section-order = [
    "future",
    "standard-library",
    "third-party",
    "first-party",
    "local-folder"
]
Docstring Checks
[tool.ruff]
select = ["D"]  # Enable docstring checks

[tool.ruff.pydocstyle]
convention = "google"  # or "numpy", "pep257"

Practical Examples

Example 1: Unused Imports

Before:

import os
import sys
import pandas as pd  # unused

Ruff will detect and can auto-remove:

$ ruff check . --fix
Fixed 1 error:
- F401 [*] `pandas` imported but unused
Example 2: Import Sorting

Before:

from myapp import utils

from typing import List

After Ruff format:

import os
import sys
from typing import List

from myapp import utils
Example 3: Code Modernization

Before (Python 3.7 style):

from typing import List

def process(items: List[str]) -> None:
    pass

After Ruff with pyupgrade:

def process(items: list[str]) -> None:
    pass

Common Ruff Commands

Command Description
ruff check . Lint all files
ruff check . --fix Lint and auto-fix issues
ruff format . Format all files
ruff format . --check Check formatting without changes
ruff check . --select F Run only Pyflakes rules
ruff check . --ignore E501 Ignore specific rule
ruff rule E501 Show details about a rule
ruff --version Check Ruff version
ruff clean Clear Ruff cache

Best Practices

  • Start gradually: Enable basic rules first (E, F, W), then add more
  • Use in CI/CD: Run Ruff checks in your pipeline to catch issues early
  • Configure per-project: Customize rules based on team preferences
  • Auto-fix in dev: Use --fix locally, check-only in CI
  • IDE integration: Set up format-on-save for instant feedback
  • Document exceptions: Use noqa comments with explanations
  • Keep updated: Ruff is actively developed, update regularly

Migration Guide

From Flake8

Most Flake8 configuration translates directly:

# Old: .flake8 or setup.cfg
[flake8]
max-line-length = 88
ignore = E203, W503
exclude = .git,__pycache__

# New: pyproject.toml
[tool.ruff]
line-length = 88
ignore = ["E203", "W503"]
exclude = [".git", "__pycache__"]
From Black + isort
# Old: multiple tools
black .
isort .

# New: just Ruff
ruff format .
ruff check . --select I --fix  # Import sorting

Troubleshooting

Too Many Errors?

Start with minimal rules and gradually enable more:

[tool.ruff]
select = ["E4", "E7", "E9", "F"]  # Critical errors only
False Positives?

Ignore specific rules or files:

[tool.ruff]
ignore = ["E501"]  # Ignore line length

[tool.ruff.per-file-ignores]
"tests/*" = ["S101"]  # Allow asserts in tests
Conflicts with Black?

Ruff's formatter is Black-compatible, but if needed:

[tool.ruff]
# Ignore rules that conflict with Black
ignore = ["E501", "W505"]
