Why Do We Need a Loop?

Most real-world tasks aren't one-shot. You ask an agent to draft something, review it, and often it needs another pass. You submit code, it fails a check, so you fix and retry. You send a summary to a quality evaluator and it comes back as "not good enough — try again."

The LoopAgent is ADK's answer to this class of problems. It runs its sub-agents in a loop, repeatedly, until either a maximum number of iterations is reached or a sub-agent explicitly signals that the loop should stop. No manual while loops, no callback chains — just a declarative loop wrapper.

When to use LoopAgent:
  • Iterative refinement — keep improving until quality threshold is met
  • Retry with feedback — failed validation triggers another attempt
  • Polling / waiting — check a condition repeatedly until it becomes true
  • Self-correcting pipelines — agent critiques its own output and rewrites

How LoopAgent Works

A LoopAgent wraps one or more sub-agents. On each iteration, it runs all sub-agents in order (exactly like a single pass of SequentialAgent). After the last sub-agent finishes, it checks two conditions:

  1. Has max_iterations been reached? If yes, stop.
  2. Did any sub-agent's tool call set tool_context.actions.escalate = True? If yes, stop.

If neither condition is true, the loop runs again from the first sub-agent. The conversation history grows with each iteration, so each pass has full context of what happened in previous passes. 


# How a loop terminates early — inside a tool function
def check_quality(score: int, tool_context: ToolContext) -> dict:
    """Check if content quality is sufficient to stop the loop.

    Args:
        score: Quality score from 1-10.
        tool_context: ADK tool context.

    Returns:
        A dict with pass/fail status. Sets escalate=True if quality passes.
    """
    if score >= 8:
        tool_context.actions.escalate = True   # signals LoopAgent to stop
        return {"status": "passed", "score": score, "action": "loop complete"}
    return {"status": "failed", "score": score, "action": "retry"}
Important: Setting tool_context.actions.escalate = True inside a tool is the way a sub-agent signals "we are done, stop looping." Without this, the loop runs until max_iterations is exhausted.

Project: Content Refinement Loop

We will build a loop that iteratively improves a piece of writing:

  • writer_agent — writes or rewrites a piece of content, saving it to state
  • critic_agent — scores the content on a scale of 1–10 and provides improvement notes. If the score reaches 8 or above, it escalates to stop the loop.

The loop runs up to max_iterations=4 times. If the critic is satisfied early, it exits before reaching the maximum.

Step 1: Define the State Tools


# agent.py
import os
from dotenv import load_dotenv
from google.adk.agents import Agent, LoopAgent
from google.adk.tools import ToolContext

load_dotenv()

# ── State tools ───────────────────────────────────────────────────────

def save_content(content: str, tool_context: ToolContext) -> dict:
    """Save the current version of the content to session state.

    Args:
        content: The written content to save.
        tool_context: ADK tool context.

    Returns:
        Confirmation dict with iteration count.
    """
    iteration = tool_context.state.get("iteration", 0) + 1
    tool_context.state["content"] = content
    tool_context.state["iteration"] = iteration
    return {"status": "saved", "iteration": iteration}


def load_content(tool_context: ToolContext) -> dict:
    """Load the current content and iteration count from session state.

    Args:
        tool_context: ADK tool context.

    Returns:
        Dict with content text and iteration number.
    """
    return {
        "content": tool_context.state.get("content", ""),
        "iteration": tool_context.state.get("iteration", 0),
    }


def submit_score(score: int, feedback: str, tool_context: ToolContext) -> dict:
    """Record a quality score and feedback. Escalates loop if score >= 8.

    Args:
        score: Quality score from 1 to 10.
        feedback: Specific improvement notes for the writer.
        tool_context: ADK tool context.

    Returns:
        Dict with score, feedback, and whether the loop is stopping.
    """
    tool_context.state["last_score"] = score
    tool_context.state["last_feedback"] = feedback

    if score >= 8:
        tool_context.actions.escalate = True
        return {
            "score": score,
            "feedback": feedback,
            "loop_status": "complete — quality threshold met",
        }
    return {
        "score": score,
        "feedback": feedback,
        "loop_status": f"retry — need score >= 8, got {score}",
    }

Key Parameters Reference

Parameter / Concept Description
max_iterations Maximum number of loop passes. Loop stops even if escalate never fires. Always set this to avoid infinite loops.
tool_context.actions.escalate Set to True inside any tool to signal the LoopAgent to stop after the current iteration completes.
tool_context.state Shared dictionary persisting across all iterations and all agents. Use it to track scores, feedback, iteration count, and content versions.
sub_agents order Within each iteration, sub-agents run in the order listed — just like SequentialAgent.
Conversation history Grows with every iteration. Agents can read previous feedback directly from the history without needing state tools.

Common Patterns

Pattern 1: Counter-based termination

Use max_iterations as a hard cap. Simple and safe — always terminates.


LoopAgent(name="my_loop", sub_agents=[...], max_iterations=5)
Pattern 2: Condition-based termination

A validator tool checks a condition and sets escalate=True when done. Can exit early before reaching max_iterations.


def validate(result: str, tool_context: ToolContext) -> dict:
    if meets_criteria(result):
        tool_context.actions.escalate = True
        return {"status": "done"}
    return {"status": "retry", "hint": "..."}
Pattern 3: Both together (recommended)

Always combine a condition with max_iterations as a safety net. This ensures the loop never runs forever even if the condition is never met.


LoopAgent(
    name="safe_loop",
    sub_agents=[generator, validator],
    max_iterations=10,   # safety cap
)
# validator calls tool_context.actions.escalate = True when done

Things to Watch Out For

  • Always set max_iterations. Without it, if escalate is never triggered, the loop runs indefinitely.
  • Growing context window. Each iteration adds to the conversation history. For many iterations (10+), this can get expensive. Use state tools to pass data instead of relying on history alone.
  • Escalate fires after the full iteration. Setting escalate=True in a tool does not stop the current agent mid-run. The rest of the current iteration completes, then the loop exits.
  • Track iteration count. Store it in state so agents know which pass they're on and can adjust their behavior (e.g., be more aggressive on the final allowed pass).

Final Thoughts

The LoopAgent transforms one-shot generation into an iterative improvement process. The combination of max_iterations as a hard cap and tool_context.actions.escalate as an early-exit condition gives you precise control over when to stop — without any manual while-loop boilerplate.

The shared state dictionary is your best friend here: use it to track scores, feedback, versions, and iteration counts so your agents have full context at every pass.
